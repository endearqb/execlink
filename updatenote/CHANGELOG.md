# ExecLink 历史版本更新记录

说明：
- 本文件按版本号整理历史更新摘要，便于查看发布演进。
- 详细实施过程、验证命令与中间决策，仍以 `updatenote/*.md` 和 `tasks/todo.md` 为准。
- 仓库当前未发现 `v0.2.0`、`v0.2.1` 对应 tag；以下版本以现有 git tag、任务记录和工作区构建记录为依据。

## v0.2.11

日期：2026-03-16
状态：已发布 GitHub Release / 已打 tag

更新摘要：
- 为 `claude`、`gemini`、`codex` 新增固定启动方式子项，支持独立显示名和开关，并同步写入右键菜单。
- 统一普通安装与快速安装向导的计时和复检规则：命令结果立即判定，复检成功立即推进，仅在持续未完成时才等到超时上限。
- 安装失败提示细化为“执行失败 / 执行超时 / 复检失败 / 复检超时”，排障路径更清晰。
- 保持升级链路为“命令成功后单次复检”，不新增额外轮询等待。

## v0.2.9

日期：2026-03-06
状态：工作区版本 / 已完成本地 MSI 与 NSIS 构建，当前仓库未见正式 tag

更新摘要：
- 右键菜单链路从 `Nilesoft + PowerShell HKCU 脚本 + shell.exe -restart` 重构为 `Rust 直写 HKCU 注册表 + v2 schema + SHChangeNotify`。
- 新增 v2 菜单内部模块：菜单模型、命令编译、registry builder、registry writer、shell notify、service。
- 彻底移除旧 Nilesoft 运行时/安装链路、Explorer 旧模块与 HKCU PowerShell 脚本辅助，打包资源不再携带 `nilesoft.zip`。
- 修复 Windows 经典级联菜单仅显示分组名不显示子项的问题：父键显式写入 `SubCommands=""`。
- 右键菜单子项改用各 CLI 品牌图标；父分组继续使用 ExecLink 图标。
- 新增 Win11 经典右键菜单显式开关，可在应用内切换“经典菜单覆盖 / 原生顶层菜单”。
- 菜单页将 `终端运行器`、`uv 安装源策略`、`安装超时` 收拢到“高级维护”，主区只保留高频菜单开关。

兼容边界：
- 默认仍采用 Windows 经典右键菜单方案。
- Windows 11 若不启用系统级经典菜单开关，仍需通过“显示更多选项”进入 ExecLink 菜单。
